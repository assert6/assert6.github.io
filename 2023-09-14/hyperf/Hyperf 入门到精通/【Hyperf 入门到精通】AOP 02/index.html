AOP 与Middleware 非常相似, 当你熟练掌握Middleware 时, 相当于也掌握了AOP

Middleware 作为解耦神器, 非常普遍的应用在各个服务中
但是, 这么好用的功能仅能作用在Controller 上, 岂不是很可惜?

AOP 可以简单理解为, 可以作用在任何Class 的Middleware:

AOP 使用场景有哪些? 可以思考Middleware 使用场景有哪些:

  • Token 鉴权
  • 签名/验签, 加密/解密

  • ….
